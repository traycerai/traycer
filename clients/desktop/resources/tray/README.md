# `resources/tray/` - packaged tray icon assets

Native tray icons consumed by `src/main/tray.ts`. Both the development run
(`bun run dev`) and packaged builds resolve from this directory; in packaged
builds `electron-builder` stages it into `process.resourcesPath/tray/` via the
`extraResources` entry in `package.json`.

## Required files

| File                  | Size       | Purpose                                                                           |
| --------------------- | ---------- | --------------------------------------------------------------------------------- |
| `trayTemplate.png`    | 16x16 RGBA | macOS template image (black + alpha; AppKit inverts it for light/dark menu bars). |
| `trayTemplate@2x.png` | 32x32 RGBA | macOS retina variant of the template image.                                       |
| `tray.png`            | 16x16 RGBA | Windows / Linux tray icon (white + alpha so it stays visible on dark trays).      |
| `tray@2x.png`         | 32x32 RGBA | Retina variant of the Windows / Linux icon.                                       |

`tray.ts` selects `trayTemplate.png` on `darwin` and calls
`nativeImage.setTemplateImage(true)`; on every other platform it loads
`tray.png` as-is. Electron auto-picks the `@2x` variant on retina displays when
`nativeImage.createFromPath()` is given the 1x path.

## Regeneration

The PNGs are generated by `scripts/generate-tray-icons.cjs` (pure `node:zlib`,
no native dependencies). Re-run the script after editing the glyph or color
values:

```bash
bun clients/desktop/scripts/generate-tray-icons.cjs
```

## Build-time precheck

`scripts/check-tray-assets.cjs` runs as `prepack:check-tray` before
`electron-builder`. It fails fast if any of the four files above is missing, so
a misnamed or absent asset cannot ship as an invisible tray.
